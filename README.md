# OCR Telegram Bot

Telegram-бот для **распознавания текста на изображениях** с поддержкой
русского и английского языков.

Проект выполнен в рамках курсовой работы по дисциплине
«Прикладное машинное обучение».
**Тема:** «Разработка программного обеспечения для распознавания текста
на изображениях с пользовательским интерфейсом Telegram-бота».

## Возможности

- Принимает изображения как **фото** и как **документ**.
- Выполняет предобработку через **OpenCV**: grayscale → шумоподавление →
  бинаризация (Otsu) → масштабирование при необходимости.
- Распознаёт текст через **EasyOCR** (русский + английский).
- Возвращает пользователю:
  - распознанный текст;
  - время обработки;
  - среднюю уверенность модели (confidence).
- Команды: `/start`, `/help`, `/about`.
- Корректно обрабатывает ошибки: не-изображение, слишком большой файл,
  пустой результат, повреждённый файл, отсутствие токена.
- Настройки — через файл `.env` (токен **не хранится** в коде).
- Логирование в консоль.

## Установка и запуск на Windows

### 1. Установка Python

1. Откройте <https://www.python.org/downloads/windows/> и скачайте
   установщик **Python 3.10 или новее** (рекомендуется 3.11).
2. Запустите установщик и **обязательно** поставьте галочку
   «Add Python to PATH», затем выберите «Install Now».
3. Проверьте установку, открыв PowerShell:

   ```powershell
   python --version
   ```

   Должна быть выведена версия `Python 3.10.x` или новее.

### 2. Получение токена Telegram-бота

1. В Telegram найдите бота [@BotFather](https://t.me/BotFather).
2. Отправьте команду `/newbot`, придумайте имя и username.
3. BotFather пришлёт токен вида `123456789:AAA...` — сохраните его.

### 3. Клонирование репозитория

```powershell
git clone https://github.com/<ваш-логин>/ocr-telegram-bot.git
cd ocr-telegram-bot
```

> Если git не установлен, скачайте его с <https://git-scm.com/download/win>.

### 4. Создание виртуального окружения

```powershell
python -m venv .venv
```

### 5. Активация виртуального окружения

```powershell
.\.venv\Scripts\Activate.ps1
```

Если PowerShell ругается на политику выполнения скриптов, разово
разрешите её для текущего пользователя:

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

После активации в начале строки приглашения появится `(.venv)`.

### 6. Установка зависимостей

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

> При первом запуске EasyOCR скачает модели (~100–200 МБ) —
> это нормально, делается один раз.

### 7. Создание файла `.env`

Скопируйте шаблон:

```powershell
copy .env.example .env
```

Откройте `.env` в любом редакторе и впишите свой токен:

```
BOT_TOKEN=123456789:AAA...ваш_токен...
```

### 8. Запуск бота

```powershell
python run.py
```

В консоли должно появиться примерно следующее:

```
... | INFO     | run | Загружены настройки. Языки OCR: ['ru', 'en']
... | INFO     | bot.main | Инициализация EasyOCR ...
... | INFO     | bot.main | Бот запущен: @your_bot_name (id=...). Режим: long polling.
```

### 9. Проверка бота в Telegram

1. Откройте Telegram и найдите своего бота по username.
2. Отправьте `/start` — должно прийти приветственное сообщение.
3. Отправьте `/help` и `/about` — проверьте, что справка работает.
4. Отправьте любое фото с текстом — бот ответит распознанным текстом,
   временем обработки и уверенностью модели.
5. Отправьте картинку **как документ** (скрепка → «Файл») — качество
   будет лучше, потому что Telegram не сжимает файл.
6. Для остановки бота нажмите `Ctrl+C` в консоли.

## Запуск тестов

```powershell
pytest -v
```

Тесты не требуют интернета и моделей EasyOCR — тяжёлые зависимости
заменяются моками.

## Демонстрация для курсовой работы

Ниже — пошаговый сценарий, который удобно показать преподавателю
и приложить к отчёту в виде скриншотов.

### Что выполнить

1. **Подготовка окружения.** Открыть PowerShell в папке проекта и
   выполнить:
   ```powershell
   .\.venv\Scripts\Activate.ps1
   python run.py
   ```
2. **Команды бота.** В Telegram отправить боту:
   - `/start`
   - `/help`
   - `/about`
3. **Распознавание фото.** Отправить картинку с печатным текстом
   как **фото**.
4. **Распознавание документа.** Отправить ту же картинку как
   **документ** (через скрепку → «Файл»). Сравнить качество —
   у документа уверенность обычно выше.
5. **Английский текст.** Отправить картинку с английским текстом.
6. **Ошибки.** Проверить обработку ошибок:
   - отправить текстовое сообщение → бот попросит изображение;
   - отправить документ `.txt` → бот скажет «работаю только с изображениями»;
   - временно убрать `BOT_TOKEN` из `.env` и запустить `python run.py` →
     в консоли появится сообщение о ConfigError.
7. **Запуск тестов.** Выполнить `pytest -v` и приложить вывод.

### Какие скриншоты сделать

| № | Скриншот | Где сделать |
|---|----------|-------------|
| 1 | Окно PowerShell с выводом `python run.py` (видно «Бот запущен») | Консоль |
| 2 | Ответ на `/start` | Telegram |
| 3 | Ответ на `/help` | Telegram |
| 4 | Ответ на `/about` | Telegram |
| 5 | Распознавание фото с русским текстом (виден текст, время, confidence) | Telegram |
| 6 | Распознавание того же изображения, отправленного как документ | Telegram |
| 7 | Распознавание английского текста | Telegram |
| 8 | Ответ бота на текстовое сообщение (ошибка «работаю только с изображениями») | Telegram |
| 9 | Ответ бота на пустое/нечитаемое изображение | Telegram |
| 10 | Запуск `pytest -v` со списком пройденных тестов | Консоль |

Готовые скриншоты удобно сохранять в папку `data/samples/` рядом
с исходными изображениями — тогда их легко вставить в текст курсовой.

## Тонкая настройка

Все параметры читаются из `.env` (см. `.env.example`):

| Переменная        | По умолчанию | Описание |
|-------------------|--------------|----------|
| `BOT_TOKEN`       | —            | Токен от @BotFather (обязательно) |
| `OCR_LANGUAGES`   | `ru,en`      | Список языков EasyOCR через запятую |
| `USE_GPU`         | `false`      | Использовать ли CUDA-GPU |
| `MAX_FILE_SIZE_MB`| `20`         | Лимит размера входящего файла |
| `KEEP_UPLOADS`    | `false`      | Сохранять ли загруженные изображения |

## Возможные проблемы

- **`ConfigError: Не задан BOT_TOKEN`** — проверьте, что в корне проекта
  лежит файл `.env` и в нём заполнена переменная `BOT_TOKEN`.
- **Первый запуск долгий** — EasyOCR скачивает модели весов.
  Дождитесь сообщения «EasyOCR готов к работе».
- **`ModuleNotFoundError`** — не активировано виртуальное окружение
  или не установлены зависимости (`pip install -r requirements.txt`).
- **Бот не отвечает** — проверьте, что токен правильный и нет ещё
  одного запущенного экземпляра бота с тем же токеном.
